---

### **STEP 2: CONFIGURE URL ROUTING**

**Option A: Static Hosting (Netlify, Vercel, Cloudflare Pages)**

Create `_redirects` file:
```
/landscaping-estimating-software-* /locations/landscaping-estimating-software-:splat 200
```

**Option B: Apache (.htaccess)**

Add to your `.htaccess` file:
```apache
RewriteEngine On
RewriteRule ^landscaping-estimating-software-([a-z-]+)$ /locations/landscaping-estimating-software-$1.html [L]
```

**Option C: Nginx**

Add to your nginx config:
```nginx
location ~ ^/landscaping-estimating-software-([a-z-]+)$ {
    try_files /locations/landscaping-estimating-software-$1.html =404;
}
```

**Option D: Node.js/Express**

```javascript
app.get('/landscaping-estimating-software-:city', (req, res) => {
    const citySlug = req.params.city;
    const filepath = path.join(__dirname, 'public', 'locations', `landscaping-estimating-software-${citySlug}.html`);
    
    if (fs.existsSync(filepath)) {
        res.sendFile(filepath);
    } else {
        res.status(404).send('City not found');
    }
});
```

- As long as local content differs (which it does), you're fine
